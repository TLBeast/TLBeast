## What is a process?

A **process** is a running program — static on disk until the OS springs it into action.

Want to run multiple programs? **Virtualize** — add **time sharing** and **space sharing** — and you get into **context switching**. **Policies** are the algorithms the OS uses to make decisions.

To understand a process, you need to understand **machine state**.

## Machine state

**Memory** is part of machine state. The **address space** is the memory a process can address.

**Registers** are another part of the process:
- **Program counter / instruction pointer** — which instruction is executing
- **Stack pointer & frame pointer** — managing the stack for params, vars, return addresses

## Process API

- **Create**
- **Destroy**
- **Wait**
- **Miscellaneous** (e.g. suspend and resume)
- **Status**

## Process creation

1. Program initially resides on **disk**
2. OS reads from disk and places it in **memory**
3. **Loading** — early OS loaded all at once; modern OS **lazy loads** using paging and swapping
4. Allocate the **run-time stack**, set up `main()`
5. Allocate memory for the **heap** — used for dynamically allocated data (`malloc()`, `free()`); as the heap grows, the OS allocates more
6. Set up **I/O** — Unix has input, output, and error file descriptors
7. Jump to program start — **`main()`**

## Process states

| State | Meaning |
|---|---|
| **Running** | Currently executing on the CPU |
| **Ready** | Can run, waiting for the scheduler |
| **Blocked** | Waiting on I/O or an event |

**Transitions:**
- Schedule: ready → running
- Deschedule: running → ready
- I/O initiate: running → blocked
- I/O done: blocked → ready

## What's next

The OS is ultimately a program — it needs **data structures** to keep track of everything. That means a **process list**, register context, and context switching.
