## Why swap?

Supporting **large address spaces** for many concurrent processes means another level in the memory hierarchy: stash less-used pages on disk.

**Swap space** = the disk region for moving pages back and forth.

- Swap **out** of memory → to disk
- Swap **in** from disk → to memory

The OS must remember each page's **disk address**. Swap size caps how many memory pages the system can hold at once.

For now: assume a **hardware-managed TLB**.

## Translation path

1. Process issues a VA → hardware extracts the **VPN**
2. **TLB hit** → physical address, done
3. **TLB miss** → use **PTBR** to find the page table, index by VPN → get the **PTE**
4. If PTE **valid** → put **PFN** in the TLB, retry (now a hit)

## Present bit & page faults

Need a **present bit**: is this page actually in physical memory?

Accessing a page that isn't → **page fault** → page fault handler.

While the fault is serviced (disk I/O in flight), the process is **blocked** — OS can run other ready processes.

If memory is full → need a **page replacement policy**.

## Decision tree (after TLB miss)

| Valid | Present | What happens |
|---|---|---|
| 1 | 1 | Pull PFN into TLB |
| 1 | 0 | Page fault handler |
| 0 | — | Trap → OS terminates the process |

## Watermarks

Swap / page daemon goal:

- **HW** (high watermark) → enough free pages available
- **LW** (low watermark) → free pages too low → start replacing
