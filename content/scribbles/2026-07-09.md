## Limited direct execution

Programs run **directly on the hardware** for efficiency — but the OS steps in at critical moments (e.g. system calls).

Three goals:

- **Efficiency** — OS stays out of the way as much as possible
- **Control** — OS maintains control over the hardware
- **Flexibility** — address spaces sized to what the program actually needs

## Hardware-based address translation

Hardware transforms a **virtual address** into a **physical address**.

Assumption for now: the address space sits **contiguously** in physical memory.

Each process gets the illusion its address space starts at **0** — the OS transparently relocates it elsewhere in physical memory.

### Base and bounds registers

Each CPU needs:

- **Base register** — the offset (where this process's address space starts in physical memory)
- **Bounds register** — the limit (how big the address space is)

**Translation:** `Physical Address = Virtual Address + Base`

### Examples

Instruction at virtual PC 128, base = 32 KB (32,768):

```
128 + 32,768 = 32,896  → physical address to fetch the instruction
```

Variable `x` at virtual address 15 KB, base = 32 KB:

```
32 KB + 15 KB = 47 KB  → physical address of x
```

### Dynamic relocation

Translation happens at **runtime** — address spaces can move even after a process has started.

The **MMU** (Memory Management Unit) in the processor handles this. The base register is essentially an offset.

### Protection

The **bounds register** checks legality — is this access within the process's address space? Out-of-bounds → exception.

## OS support points to consider

1. **Memory tracking** — OS tracks free physical memory with a **free list** (ranges of available physical memory)

2. **Privilege levels** — OS runs in **kernel mode**, apps in **user mode**. A bit in the **processor status word** tracks the current mode.

3. **Context switching** — when switching processes, save and restore each process's **base-and-bounds pair** in its **PCB** (Process Control Block)

4. **Exception handling** — OS handles exceptions when a program accesses memory outside its bounds
