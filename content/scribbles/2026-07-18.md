## The problem

Linear page tables keep **invalid regions** in memory. Multi-level page tables fix that: turn the linear table into a **tree**.

## The idea

Chop the page table into **page-sized units**. If an entire page of PTEs is invalid, **don't allocate** that page of the page table.

To track which pages of the page table exist, use a new structure: the **page directory**.

## Page directory

The page directory tells you:

- Where a page of the page table lives in memory, **or**
- That that entire page of the page table has **no valid pages**

**PDBR** (Page Directory Base Register) points at the page directory — e.g. **PFN 200**.

Each **PDE** (page directory entry) has:

| Field | Meaning |
|---|---|
| **Valid bit** | Is this page of the page table allocated? |
| **PFN** | Where that page of the page table lives |

If a PDE is valid → **≥ 1** page in that slice is valid.  
If invalid → the rest of the PDE is **undefined** (that page of the PT isn't in memory).

## Example

**Page Directory** (PFN 200):

| V | PFN |
|---|---|
| 1 | 201 |
| 0 | — |
| 0 | — |
| 1 | 204 |

Only **two** pages of the page table are marked valid — only those two reside in memory.

**Page 0 of PT** (PFN 201):

| V | PFN |
|---|---|
| 1 | 12 |
| 1 | 13 |
| 0 | — |
| 1 | 100 |

**Page 2 of PT** (PFN 204):

| V | PFN |
|---|---|
| 0 | — |
| 0 | — |
| 1 | 86 |
| 1 | 15 |

The middle chunk of the linear table **disappears**. The page directory tracks which pages of the page table are actually allocated.
