An operating system is responsible for managing a computer's physical resources and turning them into useful abstractions for programs. Instead of forcing every program to directly control the CPU, memory, storage devices, and input/output hardware, the operating system acts as a layer between the hardware and the applications. Its main job is to make the system easier to use, more efficient, and safer.

The three major ideas behind operating systems are **virtualization**, **concurrency**, and **persistence**.

## Virtualization

Virtualization is the process of making one physical resource appear like multiple separate resources.

### CPU virtualization

A computer may only have a limited number of physical CPUs or cores, but the operating system gives each running program the illusion that it has its own CPU. It does this by rapidly switching between programs using scheduling and context switching. This makes it seem like many programs are running at once, even though a single CPU core can only execute one stream of instructions at a time.

On systems with multiple cores, some programs or threads can truly run in parallel, but the operating system still manages which tasks get access to the CPU and when.

### Memory virtualization

Memory virtualization works in a similar way. Each program needs memory to store instructions, variables, data, and dynamically allocated objects. However, programs do not directly use physical memory addresses. Instead, the operating system gives each process its own virtual address space.

This makes each process feel like it has a private memory area, even though the actual physical memory is shared across many processes. For example, when a program uses `malloc()`, it requests memory from the heap, and the address it receives is part of its virtual address space. This abstraction allows the operating system to isolate programs from one another, protect memory, and manage resources more safely.

## Concurrency

Concurrency means that multiple tasks are active during overlapping periods of time. This does not always mean they are literally executing at the exact same instant. If there is only one CPU core, the operating system may switch between tasks very quickly, giving the appearance that they are running at the same time.

True simultaneous execution is called **parallelism** and usually requires multiple CPU cores.

Concurrency becomes especially important in multithreaded programs, where a single process can create multiple threads. A **thread** is a unit of execution within a process. Threads in the same process share the same address space — meaning they can access the same code, heap, and global data — but each thread usually has its own stack and execution state.

Because threads can share memory, concurrency introduces new problems. If two threads access and modify the same data at the same time, the result can become unpredictable. This is where **atomicity** becomes important.

An atomic operation is an operation that happens as one indivisible unit. Other threads cannot observe it halfway completed. For example, increasing a counter may look like one simple operation, but at the machine level it may involve loading the value, adding one, and storing the result back. If two threads do this at the same time without synchronization, they may interfere with each other and produce the wrong result.

Operating systems and programming libraries use tools such as locks and synchronization mechanisms to help prevent these race conditions.

## Persistence

Main memory, such as DRAM, is **volatile** — its contents disappear when the power is turned off. However, users and programs need data to last beyond a single program execution or computer restart.

To solve this, computers use persistent storage devices such as hard drives and solid-state drives. The operating system provides a **file system**, which organizes persistent data into files and directories. Programs interact with the file system through system calls such as `open()`, `read()`, `write()`, and `close()`. These system calls allow programs to store and retrieve data without needing to directly control the storage hardware.

The operating system also uses **device drivers** to communicate with hardware devices. A device driver is software that translates general operating system requests into commands that a specific device can understand. For example, instead of every program needing to know the exact details of how an SSD works, the operating system and its drivers handle those details.

File systems also use techniques such as **journaling** and **copy-on-write** to protect data. These methods carefully manage the order of writes to disk so that the file system can recover correctly if the computer crashes or loses power in the middle of an operation.

## Protection

Protection is another key role of the operating system. Normal programs run in **user mode**, where they have limited privileges. This prevents them from directly accessing hardware, modifying arbitrary memory, or interfering with the operating system itself.

When a program needs a privileged service — such as reading a file or requesting input/output — it makes a **system call**. A system call causes a trap into the operating system, switching the CPU into **kernel mode**. In kernel mode, the operating system can safely perform the requested operation. Once the operation is complete, the system performs a return from trap and switches back to user mode.

This separation between user mode and kernel mode helps provide security, isolation, and stability.

## Putting it together

The operating system takes complex physical hardware and turns it into cleaner, safer, and more manageable abstractions.

- Through **virtualization**, it makes limited resources like the CPU and memory appear more flexible and private.
- Through **concurrency**, it allows many tasks and threads to make progress at the same time while managing the challenges of shared data.
- Through **persistence**, it ensures that important information can survive even after a program ends or the computer shuts down.

These ideas work together to make modern computing systems efficient, reliable, and usable.
