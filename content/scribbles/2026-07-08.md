## Stack vs heap

| | Stack | Heap |
|---|---|---|
| Also called | **Automatic memory** | **Long-lived memory** |
| Managed by | Compiler (implicit) | Programmer (explicit) |

## malloc()

`malloc()` takes a **size** and requests space on the heap.

Use `sizeof()` to request the right amount:

```c
double *d = malloc(sizeof(double));
int *arr = malloc(10 * sizeof(int));  // array of 10 ints
```

**Note:** `sizeof()` returns the size of a pointer to an int, not the total dynamically allocated memory — that's why you multiply by the count.

**Strings:** use `malloc(strlen(s) + 1)` instead of `sizeof()` — you need room for the null terminator.

## free()

Release heap memory when you're done:

```c
int *x = malloc(10 * sizeof(int));
free(x);
```

## Common memory bugs

1. **Forgetting to allocate** — `strcpy(dst, src)` segfaults if `dst` was never allocated. Fix: `char *dst = (char *) malloc(strlen(src) + 1);`

2. **Not allocating enough** — forgetting the `+1` for the null terminator on strings

3. **Forgetting to initialize** — **uninitialized read**: the program reads an unknown value from the heap

4. **Memory leak** — forgetting to `free()`

5. **Dangling pointer** — freeing memory before the program is done using it

6. **Double free** — calling `free()` on the same address twice → crash

7. **Invalid free** — calling `free()` on something that wasn't malloc'd
