## Beyond two levels

So far: page directory + pieces of the page table. A **deeper tree** is possible when the directory itself gets too big.

## Setup

| | |
|---|---|
| VA | **30 bits** |
| Page size | **512 bytes** |
| Offset | **9 bits** (`2⁹ = 512`) |
| VPN | **21 bits** (`30 − 9`) |
| PTE size | **4 bytes** |

**Goal:** every piece of the page table fits in a **single page**.

## How many levels?

PTEs per page: `512 / 4 = 128` → need **7 bits** of VPN to index one page.

Two-level attempt:

```
29 … 16 | 15 … 9 | 8 … 0
 PD idx  | PT idx | offset
 14 bits | 7 bits | 9 bits
```

Bits **16:29** = 14 bits → `2¹⁴` directory entries → spans **128 pages**, not one. Directory doesn't fit.

## Fix: split the directory

Chop the page directory into pages, then put **another page directory** on top to point at them:

```
29 … 23 | 22 … 16 | 15 … 9 | 8 … 0
PDIndex0 | PDIndex1 | PT idx | offset
  7 bits |  7 bits  | 7 bits | 9 bits
```

Three levels. Every index is 7 bits → every table piece fits in one page.

## Hardware path

1. Check the **TLB**
2. **Hit** → form the physical address directly (no page-table walk)
3. **Miss** → full multi-level lookup

## Aside

**Inverted page tables** = one entry per **physical** page of the system (single table for the whole machine).
