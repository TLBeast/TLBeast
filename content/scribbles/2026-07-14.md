## Variable-sized vs fixed-size

The OS manages space either with **variable-sized pieces** or **fixed-size pieces**.

Variable-sized chunks can become **fragmented**. The fixed-size approach leads to **paging**.

## Pages and page frames

- Each fixed-size unit of the address space is a **page**
- Each slot of **physical memory** (not the address space) is a **page frame**
- The OS keeps a **free list** of available pages

## Address translation

A virtual address splits into:

- **VPN** (virtual page number) — index into the page table
- **Offset** — which byte within the page (not translated)

The **page table** stores address translations, e.g. `VP1 → PF7`.

Look up the VPN in the page table → get the **PFN / PPN** (physical frame / page number). Stick the offset on → physical address.

## Page tables get huge

A 20-bit VPN means up to **2²⁰** translations. Page tables are too big to stash in special on-chip MMU hardware for the current process — they live in **main memory**.

A page table is just a data structure mapping virtual addresses to physical ones.

### Linear page table

Simple array: OS indexes by VPN, looks up the **PTE** at that index, and finds the PFN.

## PTE bits

Each **page table entry** carries control bits:

| Bit | Meaning |
|---|---|
| **Valid** | Is this translation valid? Unused space is marked invalid |
| **Protection** | Can the page be read / written / executed? |
| **Present** | Is the page in memory, or swapped out to disk? |
| **Dirty** | Has the page been modified since it was brought into memory? |
| **Reference** | Has the page been accessed recently? |
