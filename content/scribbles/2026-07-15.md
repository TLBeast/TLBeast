## What is a TLB?

The **TLB** (Translation Lookaside Buffer) lives in the chip's **MMU**. It's a hardware cache of popular **virtual → physical** address translations — so you don't always have to walk the page table in main memory.

## TLB hit

1. Hardware extracts the **VPN** from the virtual address
2. Checks the TLB for that translation
3. Found → **TLB hit**
4. Pull the **PFN** from the TLB entry, concatenate with the **offset** → physical address

## TLB miss

Translation not in the TLB → **TLB miss**.

Hardware consults the **page table**, then **updates the TLB** with the new translation.

## Hit rate

**TLB hit rate** = (number of hits) / (total number of accesses)

Goal: hit rates approaching **100%**. That relies on **temporal locality** and **spatial locality**.

## Hardware shape

A typical TLB is a **fully associative cache** — any translation can live in any entry (often 32, 64, or 128 entries).

Because it's fully associative, the hardware searches the **entire TLB in parallel** for a match.

## TLB entry

Classic entry: `VPN | PFN | other bits`

Both VPN and PFN are needed in each entry because of the fully associative layout.

| Bit | Meaning |
|---|---|
| **Valid** | Does this entry hold a valid translation? |
| **Protection** | How can the page be accessed (read / write / execute)? |
| **Address space identifier** | Which address space this translation belongs to |
| **Dirty** | Has the page been modified? |
